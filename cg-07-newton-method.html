PAGE 07; PRE-MIDTERM

Newton's Method for Optimization

Use curvature to build a local bowl model, then jump to its bottom; the fastest local method

Part A: The Idea; Build a Local Bowl, Jump to Its Bottom

Physical analogy; From feeling the slope to building a model.

Remember gradient descent from Page 06? You are blindfolded on a mountain and all you can do is feel the slope under your feet. You take a step in the steepest downhill direction. That is first-order information; the gradient tells you the slope.

Now imagine you have a superpower: not only can you feel the slope, but you can also feel the curvature of the ground. You can tell whether the slope is getting steeper, flattening out, or curving back up. With this extra information, you can build a little bowl-shaped model of the ground around your feet. It is like sculpting a small bowl out of clay that matches the slope and curvature of the real ground where you stand.

Once you have this bowl model, you know exactly where the bottom of the bowl is. So instead of taking a cautious step downhill, you jump directly to the bottom of the bowl. If the bowl model accurately represents the actual terrain, you land right at the minimum (or very close to it).

That is Newton's method: measure both slope (gradient) and curvature (Hessian), build a quadratic bowl model, jump to its bottom. The key insight is that Newton uses second-order information (the Hessian = curvature), while gradient descent only uses first-order information (the gradient = slope). This extra information is what makes Newton so much faster; but also more expensive and sometimes unreliable far from the solution.
The Quadratic Model at Point $x_k$ $$m_k(p) = f(x_k) + \nabla f(x_k)^T p + \frac{1}{2} p^T \nabla^2 f(x_k)\, p$$
$m_k(p)$
the model; a quadratic function that approximates $f$ near $x_k$. The variable $p$ represents a step away from $x_k$.
$f(x_k)$
the current function value; the "height" at your current position. This is a constant (just a number).
$\nabla f(x_k)^T p$
the slope term; how much the function changes as you move by $p$, according to the linear approximation. This is the gradient (from Page 02) dotted with the step.
$\frac{1}{2} p^T \nabla^2 f(x_k)\, p$
the curvature term; how the slope itself changes as you move by $p$. This uses the Hessian matrix $\nabla^2 f(x_k)$ (from Page 02).

This is just the second-order Taylor expansion of $f$ around $x_k$ (recall Taylor's theorem from Page 01). For a general function, this model is an approximation. For a quadratic function, this model is exact.

To find the bottom of the bowl model, we take the gradient of $m_k$ with respect to $p$ and set it to zero:

Derivation: The Newton Step

Step 1: Write the model
$$m_k(p) = f(x_k) + \nabla f(x_k)^T p + \frac{1}{2} p^T \nabla^2 f(x_k)\, p$$
Step 2: Differentiate with respect to $p$

Using matrix calculus rules (the gradient of $a^Tp$ is $a$, and the gradient of $\frac{1}{2}p^TBp$ is $Bp$ when $B$ is symmetric):

$$\nabla_p \, m_k(p) = \nabla f(x_k) + \nabla^2 f(x_k)\, p$$
Step 3: Set to zero and solve
$$\nabla f(x_k) + \nabla^2 f(x_k)\, p = 0$$ $$\nabla^2 f(x_k)\, p = -\nabla f(x_k)$$

$$p_k = -\left[\nabla^2 f(x_k)\right]^{-1} \nabla f(x_k)$$

Step 4: The Newton update
$$x_{k+1} = x_k + p_k = x_k - \left[\nabla^2 f(x_k)\right]^{-1} \nabla f(x_k)$$
In practice, we NEVER compute $[\nabla^2 f]^{-1}$ explicitly. Instead, we solve the linear system $\nabla^2 f(x_k) \, p = -\nabla f(x_k)$ for $p$. This is much cheaper ($O(n^3)$ vs $O(n^3)$ for inversion, but with a smaller constant).
The Newton Step (Summary)

Solve the Newton system:

$$\nabla^2 f(x_k) \, p_k = -\nabla f(x_k)$$

Then update:

$$x_{k+1} = x_k + p_k$$
$\nabla^2 f(x_k)$
the Hessian matrix at $x_k$; an $n \times n$ matrix of second partial derivatives. Captures the curvature. (recall from Page 02)
$\nabla f(x_k)$
the gradient at $x_k$; the vector of first partial derivatives. Captures the slope.
$p_k$
the Newton step; the direction and distance to move. Unlike GD, the step has a "built-in" step size (no separate $\alpha$ needed for the pure method).

GD vs Newton; The Key Difference

Gradient Descent: $p_k = -\alpha_k \nabla f(x_k)$. Uses only the gradient. Needs a separate step size $\alpha_k$. Treats all directions equally.

Newton: $p_k = -[\nabla^2 f(x_k)]^{-1} \nabla f(x_k)$. Uses both gradient and Hessian. The Hessian inverse plays the role of a "variable-specific step size"; it stretches and rotates the gradient to account for different curvatures in different directions.

Think of it this way: GD gives every variable the same step size. Newton gives each variable a step size proportional to how curved the function is in that direction. Variables with high curvature get small steps (to avoid overshooting), and variables with low curvature get big steps (to make progress faster).

Algorithm: Newton's Method (Pure / Local)
Input: starting point $x_0$, tolerance $\epsilon \gt 0$
Step 0: Set $k = 0$
Step 1: Compute the gradient $g_k = \nabla f(x_k)$ ← the slope
Step 2: If $\|g_k\| \lt \epsilon$, STOP
Step 3: Compute the Hessian $H_k = \nabla^2 f(x_k)$ ← the curvature
Step 4: Solve $H_k p_k = -g_k$ for $p_k$ ← find the bottom of the local bowl
Step 5: Update: $x_{k+1} = x_k + p_k$ ← jump to the bottom
Step 6: Set $k = k + 1$, go to Step 1

Part B: One-Step Convergence on Quadratics

The most dramatic demonstration of Newton's power: on a quadratic function, Newton converges in exactly one step from any starting point. Compare that to gradient descent, which might take dozens or hundreds of steps on the same problem!

Why Newton Converges in One Step on Quadratics

For $f(x) = \frac{1}{2}x^T Q x + b^T x + c$ (where $Q$ is symmetric positive definite):

  • Gradient: $\nabla f(x) = Qx + b$
  • Hessian: $\nabla^2 f(x) = Q$ , this is constant (does not depend on $x$)!

The Newton step from $x_0$ is:

$$p_0 = -Q^{-1}(Qx_0 + b) = -x_0 - Q^{-1}b$$

So:

$$x_1 = x_0 + p_0 = x_0 - x_0 - Q^{-1}b = -Q^{-1}b$$

But the minimum of $f$ is at $x^* = -Q^{-1}b$ (set $\nabla f = Qx + b = 0$, solve for $x$).

Therefore $x_1 = x^*$ exactly. One step. Done.

Why does this work?

The quadratic model $m_k(p)$ is an exact representation of $f$ when $f$ is quadratic. The model IS the function. So the bottom of the model IS the bottom of the function. Newton jumps directly there.

For non-quadratic functions, the quadratic model is only an approximation. The approximation is good nearby (hence Newton works fast when close to the solution) but can be bad far away (hence Newton can fail when far from the solution).

Worked Example: Newton on a Quadratic (Same Setup as GD on Page 06)

Let $Q = \begin{bmatrix} 2 & 0 \\ 0 & 3 \end{bmatrix}$, $b = \begin{bmatrix} 0 \\ 0 \end{bmatrix}$, so $f(x) = \frac{1}{2}x^TQx = x_1^2 + \frac{3}{2}x_2^2$. Start at $x_0 = \begin{bmatrix} 1 \\ 1 \end{bmatrix}$.

Step 1: Compute the gradient
$$g_0 = Qx_0 + b = \begin{bmatrix} 2 & 0 \\ 0 & 3 \end{bmatrix}\begin{bmatrix} 1 \\ 1 \end{bmatrix} = \begin{bmatrix} 2 \\ 3 \end{bmatrix}$$
Step 2: Compute the Hessian
$$H_0 = Q = \begin{bmatrix} 2 & 0 \\ 0 & 3 \end{bmatrix}$$

For a quadratic, the Hessian is just $Q$, constant everywhere.

Step 3: Solve $H_0 p_0 = -g_0$
$$\begin{bmatrix} 2 & 0 \\ 0 & 3 \end{bmatrix} p_0 = -\begin{bmatrix} 2 \\ 3 \end{bmatrix} = \begin{bmatrix} -2 \\ -3 \end{bmatrix}$$

Since $H_0$ is diagonal, the system decouples:

$$2p_1 = -2 \implies p_1 = -1$$ $$3p_2 = -3 \implies p_2 = -1$$ $$p_0 = \begin{bmatrix} -1 \\ -1 \end{bmatrix}$$
Step 4: Update
$$x_1 = x_0 + p_0 = \begin{bmatrix} 1 \\ 1 \end{bmatrix} + \begin{bmatrix} -1 \\ -1 \end{bmatrix} = \begin{bmatrix} 0 \\ 0 \end{bmatrix} = x^*$$

Newton reached the exact minimum in ONE step!

Compare: on Page 06, gradient descent with the same starting point gave $x_1 = [9/35, -4/35]$ and $x_2 = [6/175, 6/175]$; still not at the minimum after two steps. Newton got there instantly.
The price of this speed: Newton needed to compute the Hessian (a $2 \times 2$ matrix here, but $n \times n$ in general) AND solve a linear system. For large problems ($n$ = millions), this cost is prohibitive. That is what motivates quasi-Newton methods like BFGS (Page 12); they approximate the Hessian cheaply.

Part C: Theorem 3.5; Local Quadratic Convergence

Physical analogy; A GPS that only works within city limits.

Newton's method is like a GPS that gives perfect directions; but only when you are already within city limits. If you are close to the destination, the GPS is spectacularly fast: each instruction doubles the number of correct digits in your position. After a few instructions you are essentially there.

But if you are out in the countryside, far from the city, the GPS might give nonsensical directions that take you in the wrong direction or even farther away. This is the "local" in "local convergence"; Newton only guarantees convergence when you start close enough to the solution.

"Close enough" depends on the function. For well-behaved functions with smooth curvature, the "city limits" are large. For wild, rapidly-changing functions, the "city limits" might be tiny. This is where the "Lipschitz continuous Hessian" condition comes in; it guarantees the curvature does not change too wildly.
Theorem 3.5: Quadratic Convergence of Newton's Method

Suppose:

  1. (Smooth Hessian) The Hessian $\nabla^2 f$ is Lipschitz continuous: there exists $L \gt 0$ such that $\|\nabla^2 f(x) - \nabla^2 f(y)\| \leq L\|x - y\|$ for all $x, y$ near $x^*$.
  2. (Non-degenerate minimum) $\nabla f(x^*) = 0$ and $\nabla^2 f(x^*)$ is positive definite (i.e., $x^*$ is a strong local minimum).
  3. (Close enough start) $x_0$ is sufficiently close to $x^*$.

Then Newton's method converges quadratically:

$$\|x_{k+1} - x^*\| \leq C \|x_k - x^*\|^2$$

where $C$ is a constant depending on $L$ and $\nabla^2 f(x^*)$.

Let us unpack every condition in plain English:

What each condition means

Condition 1: "Lipschitz continuous Hessian"; The curvature of the function changes smoothly and gradually, not in sudden jumps. Imagine driving on a road: the road curves, but the curvature changes gradually (smooth road) rather than suddenly (right angle). The constant $L$ measures how fast the curvature can change. Smaller $L$ means smoother curvature and a bigger "safe zone" around the solution.

Condition 2: "Positive definite Hessian at $x^*$"; The solution $x^*$ is a genuine bowl-bottom, not a saddle point or flat area. This ensures the Hessian is invertible at the solution (recall from Page 03 that positive definite matrices are always invertible).

Condition 3: "Close enough start"; You must already be near the solution. How near? Roughly, within a region where the quadratic model is a good approximation of $f$. This is the "city limits" from the GPS analogy.

What "quadratic convergence" means in practice

$\|x_{k+1} - x^*\| \leq C \|x_k - x^*\|^2$ means the error gets squared each iteration (up to a constant). If you have 2 digits of accuracy, next step you have 4. Then 8. Then 16. The number of correct digits approximately doubles each step.

Iteration $k$Error $\|x_k - x^*\|$Correct digits
0$0.1$1
1$0.01$2
2$0.0001$4
3$0.00000001$8
4$10^{-16}$ (machine precision)16

Compare to gradient descent (linear convergence from Page 06): GD adds a fixed fraction of a digit each step. Newton doubles digits each step. For high precision, Newton is astronomically faster.

Exam Pattern: "Compare convergence rates"

You may be asked to compare GD (linear convergence) vs Newton (quadratic convergence). Key contrast:

  • GD: $\|e_{k+1}\| \leq c \|e_k\|$ with $c \lt 1$. Error decreases by a constant factor each step. Needs many iterations for high precision.
  • Newton: $\|e_{k+1}\| \leq C \|e_k\|^2$. Error gets squared each step. Very few iterations for high precision, but only works near the solution and costs more per iteration.

Part D: Newton on a General (Non-Quadratic) Function

On a general function, Newton does NOT converge in one step. But it is still much faster than GD when you are close to the solution. Here is a hand computation on a non-quadratic function.

Worked Example: Newton on $f(x_1, x_2) = x_1^2 + x_1 x_2 + x_2^2 - 3x_1$

Step 0: Setup
$$f(x_1, x_2) = x_1^2 + x_1 x_2 + x_2^2 - 3x_1$$

Starting point: $x_0 = \begin{bmatrix} 0 \\ 0 \end{bmatrix}$

Step 1: Compute partial derivatives
$$\frac{\partial f}{\partial x_1} = 2x_1 + x_2 - 3 \qquad \frac{\partial f}{\partial x_2} = x_1 + 2x_2$$
Step 2: Compute gradient at $x_0 = [0, 0]^T$
$$g_0 = \nabla f(x_0) = \begin{bmatrix} 2(0) + 0 - 3 \\ 0 + 2(0) \end{bmatrix} = \begin{bmatrix} -3 \\ 0 \end{bmatrix}$$
Step 3: Compute the Hessian

Second partial derivatives:

$$\frac{\partial^2 f}{\partial x_1^2} = 2, \quad \frac{\partial^2 f}{\partial x_1 \partial x_2} = 1, \quad \frac{\partial^2 f}{\partial x_2^2} = 2$$ $$H = \nabla^2 f = \begin{bmatrix} 2 & 1 \\ 1 & 2 \end{bmatrix}$$

Note: for this function the Hessian is constant (it is actually a quadratic, but written in expanded form). In general, the Hessian would depend on $x$.

Step 4: Solve $H p_0 = -g_0$
$$\begin{bmatrix} 2 & 1 \\ 1 & 2 \end{bmatrix} \begin{bmatrix} p_1 \\ p_2 \end{bmatrix} = -\begin{bmatrix} -3 \\ 0 \end{bmatrix} = \begin{bmatrix} 3 \\ 0 \end{bmatrix}$$

Using the 2x2 inverse formula: $A^{-1} = \frac{1}{\det(A)}\begin{bmatrix} d & -b \\ -c & a \end{bmatrix}$ where $A = \begin{bmatrix} a & b \\ c & d \end{bmatrix}$.

$\det(H) = 2 \cdot 2 - 1 \cdot 1 = 3$

$$H^{-1} = \frac{1}{3}\begin{bmatrix} 2 & -1 \\ -1 & 2 \end{bmatrix}$$ $$p_0 = H^{-1}\begin{bmatrix} 3 \\ 0 \end{bmatrix} = \frac{1}{3}\begin{bmatrix} 2 & -1 \\ -1 & 2 \end{bmatrix}\begin{bmatrix} 3 \\ 0 \end{bmatrix} = \frac{1}{3}\begin{bmatrix} 6 \\ -3 \end{bmatrix} = \begin{bmatrix} 2 \\ -1 \end{bmatrix}$$
Step 5: Update
$$x_1 = x_0 + p_0 = \begin{bmatrix} 0 \\ 0 \end{bmatrix} + \begin{bmatrix} 2 \\ -1 \end{bmatrix} = \begin{bmatrix} 2 \\ -1 \end{bmatrix}$$
Step 6: Verify; is this the minimum?
$$\nabla f(x_1) = \begin{bmatrix} 2(2) + (-1) - 3 \\ 2 + 2(-1) \end{bmatrix} = \begin{bmatrix} 4 - 1 - 3 \\ 2 - 2 \end{bmatrix} = \begin{bmatrix} 0 \\ 0 \end{bmatrix}$$

The gradient is exactly zero! $x^* = [2, -1]^T$ is the minimum. Newton converged in ONE step because this function is quadratic (even though it does not look like the standard form $\frac{1}{2}x^TAx$).

You can verify: $f(2, -1) = 4 - 2 + 1 - 6 = -3$. Try any nearby point: $f(2.1, -1) = 4.41 - 2.1 + 1 - 6.3 = -2.99$. Indeed $f(2,-1) = -3$ is the minimum.

Worked Example: Newton on a Truly Non-Quadratic Function

Let $f(x) = e^x - 2x$ in one dimension. We want to minimize this.

Step 0: Derivatives
$$f'(x) = e^x - 2, \qquad f''(x) = e^x$$

The minimum is where $f'(x) = 0$: $e^x = 2$, so $x^* = \ln 2 \approx 0.6931$.

Step 1: Start at $x_0 = 0$
$$f'(0) = e^0 - 2 = 1 - 2 = -1, \qquad f''(0) = e^0 = 1$$ $$p_0 = -\frac{f'(0)}{f''(0)} = -\frac{-1}{1} = 1$$ $$x_1 = 0 + 1 = 1$$
Step 2: From $x_1 = 1$
$$f'(1) = e - 2 \approx 0.7183, \qquad f''(1) = e \approx 2.7183$$ $$p_1 = -\frac{0.7183}{2.7183} = -0.2642$$ $$x_2 = 1 - 0.2642 = 0.7358$$
Step 3: From $x_2 = 0.7358$
$$f'(0.7358) = e^{0.7358} - 2 \approx 2.0871 - 2 = 0.0871, \qquad f''(0.7358) \approx 2.0871$$ $$p_2 = -\frac{0.0871}{2.0871} = -0.0417$$ $$x_3 = 0.7358 - 0.0417 = 0.6941$$
Step 4: From $x_3 = 0.6941$
$$f'(0.6941) \approx 0.0020, \qquad f''(0.6941) \approx 2.0020$$ $$p_3 \approx -0.0010$$ $$x_4 \approx 0.6931$$

After 4 iterations: $x_4 \approx 0.6931 \approx \ln 2 = x^*$. Four correct digits.

Notice the error pattern: $|x_k - x^*|$ was roughly $0.69, 0.31, 0.043, 0.001, 0.000001$; the number of correct digits roughly doubles each step after we get close. That is quadratic convergence in action!

Part E: Limitations and Why We Need More

Limitation 1: Cost of the Hessian

The Hessian matrix $\nabla^2 f(x)$ has $n^2$ entries (or $n(n+1)/2$ unique entries since it is symmetric). Computing all of them can be very expensive. Solving the $n \times n$ linear system $Hp = -g$ costs $O(n^3)$ operations.

For a problem with $n = 1{,}000$ variables: the Hessian has 500,500 unique entries, and solving the system requires about $10^9$ operations. For $n = 1{,}000{,}000$ (common in machine learning): completely infeasible.

Fix: Quasi-Newton methods like BFGS (Page 12) and SR1 (Page 13) approximate the Hessian from gradient differences, avoiding the need to compute second derivatives.

Limitation 2: Hessian might not be positive definite

If $\nabla^2 f(x_k)$ is not positive definite (e.g., the function has a saddle shape at $x_k$), the Newton step $p_k$ might point uphill; toward a maximum or saddle point, not a minimum. The quadratic model $m_k(p)$ would be a saddle or dome, not a bowl, so "jumping to the bottom" is meaningless.

Fix: Modified Newton methods add a correction to make the Hessian positive definite. Trust-region methods (Page 09) handle this naturally by restricting the step size.

Limitation 3: Only locally convergent

Theorem 3.5 requires starting "close enough." If you start far from the solution, Newton can diverge, oscillate, or converge to a wrong stationary point. The pure Newton method has no globalization; no guarantee of making progress from a distant starting point.

Fix: Line search (Page 08) and trust regions (Page 09) provide globalization. They modify the Newton step to ensure sufficient decrease in $f$ at every iteration, turning Newton from a "locally perfect" method into a "globally reliable" one.

Glossary

Newton's Method (for Optimization)
An iterative method that uses both gradient and Hessian to build a quadratic model at each point and jumps to the model's minimum. The step solves $\nabla^2 f(x_k) p_k = -\nabla f(x_k)$.
Quadratic Model $m_k(p)$
The second-order Taylor approximation of $f$ around $x_k$: $m_k(p) = f(x_k) + \nabla f(x_k)^Tp + \frac{1}{2}p^T\nabla^2 f(x_k)p$. Newton minimizes this model at each step.
Newton Step $p_k$
$p_k = -[\nabla^2 f(x_k)]^{-1}\nabla f(x_k)$. The direction and distance to the bottom of the local quadratic model.
Newton System
The linear system $\nabla^2 f(x_k) p_k = -\nabla f(x_k)$ that must be solved to find the Newton step. Solving this costs $O(n^3)$.
Quadratic Convergence
$\|x_{k+1} - x^*\| \leq C\|x_k - x^*\|^2$. The error gets squared each step, so the number of correct digits approximately doubles each iteration.
Local Convergence
The method only guarantees convergence when started "close enough" to the solution. Far from the solution, Newton may diverge.
Lipschitz Continuous Hessian
$\|\nabla^2 f(x) - \nabla^2 f(y)\| \leq L\|x - y\|$. The curvature changes smoothly; no sudden jumps. Required for Theorem 3.5.
Globalization
Strategies (line search, trust region) that modify Newton's method to guarantee progress from any starting point, not just nearby points.
