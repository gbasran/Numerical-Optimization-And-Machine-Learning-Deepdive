Logistic Regression

From continuous predictions to yes/no probabilities; the sigmoid squasher, the logistic loss, and how gradient descent, BFGS, and nonlinear CG each attack a convex but not strongly convex objective

PART A

From Regression to Classification

Analogy: the pass/fail thermometer. In linear regression (Page 23), the output is a continuous number: temperature in degrees, price in dollars, any real value on the number line. But classification asks a yes/no question: pass or fail? spam or not spam? tumor or healthy? We need to squash any real-valued score into a number between 0 and 1 that we can interpret as a probability.

The sigmoid function is that squasher. Think of it as a thermometer that reads any temperature but displays it as a percentage between "definitely cold" (0%) and "definitely hot" (100%). A raw score of 0 maps to 50% (total uncertainty). Large positive scores map close to 100%. Large negative scores map close to 0%. No matter how extreme the input, the output always stays between 0 and 1; it never breaks the thermometer.

Where linear regression (Page 23) predicts a number, logistic regression predicts a probability.

The setup: we have $n$ data points, each with $d$ features $x_i \in \mathbb{R}^d$ and a label $y_i$ indicating which class it belongs to. The model computes a raw score $\theta^T x_i$ (a dot product, exactly like linear regression) and then passes it through the sigmoid function to get a probability. Everything that follows is about choosing $\theta$ so those probabilities match the observed labels as closely as possible.

PART B

The Sigmoid Function

Analogy: a dimmer switch. Imagine a light switch that is not just on/off but has a smooth dial. Turning the knob far to the left gives you 0% brightness (off). Far to the right gives 100% brightness (full). The middle position gives exactly 50%. The transition between dim and bright is smooth, not abrupt. The sigmoid function is that dimmer switch for probabilities: it converts any real number into a smooth value between 0 and 1.

The Sigmoid Function

$$\sigma(z) = \frac{1}{1 + e^{-z}}$$
$z$
The raw score (also called the "logit"). In logistic regression, $z = \theta^T x$, the dot product of the parameter vector with the feature vector. This is the same linear combination that appears in linear regression.
$e^{-z}$
The exponential decay term. When $z$ is large and positive, $e^{-z} \approx 0$, so $\sigma(z) \approx 1$. When $z$ is large and negative, $e^{-z}$ is huge, so $\sigma(z) \approx 0$.
Output range
Always strictly between 0 and 1. Never exactly reaches either endpoint.
$\sigma(0) = 0.5$
The uncertain midpoint. When the raw score is zero, the model is perfectly undecided.
$\sigma(z) \to 1$
As $z \to +\infty$; strong positive evidence pushes the probability toward certainty.
$\sigma(z) \to 0$
As $z \to -\infty$; strong negative evidence pushes the probability toward zero.

The model for a single data point is:

$$P(y = 1 \mid x) = \sigma(\theta^T x)$$

Key properties of the sigmoid:

  • Symmetry: $\sigma(-z) = 1 - \sigma(z)$. Flipping the sign of $z$ swaps the probability between the two classes. This is why the margin-based formulation (Part C) works so cleanly.
  • Derivative: $\sigma'(z) = \sigma(z)(1 - \sigma(z))$. The derivative is largest at $z = 0$ (where $\sigma'(0) = 0.25$) and goes to zero as $|z|$ grows. In other words, the sigmoid is steepest in the middle and flattens out at the extremes; points near the decision boundary are the most "sensitive" to changes in $\theta$.
PART C

Two Label Conventions

Analogy: two ways to score a game. Convention 1 uses 0 and 1, like soccer goals: you either scored or you did not. Convention 2 uses $-1$ and $+1$, like tug-of-war: you pull left or right. Both describe the same game; they just use different number systems. The margin-based convention ($\{-1,+1\}$) turns out to be cleaner for optimization because the loss function collapses into a single, elegant formula instead of a two-part expression with an "if" baked inside.

Convention 1: $y_i \in \{0, 1\}$ (Probabilistic / Cross-Entropy)

$$\ell_i(\theta) = -\bigl[y_i \log \sigma(\theta^T x_i) + (1 - y_i)\log(1 - \sigma(\theta^T x_i))\bigr]$$
$y_i \in \{0,1\}$
The label is 0 ("negative") or 1 ("positive"). Only one of the two log-terms is active at a time: when $y_i = 1$, the $(1-y_i)$ term vanishes; when $y_i = 0$, the $y_i$ term vanishes.
$\sigma(\theta^T x_i)$
The model's predicted probability that $y_i = 1$.
$-\log(\cdot)$
The negative log turns a probability (between 0 and 1) into a loss (between 0 and $\infty$). High predicted probability for the correct class gives low loss; low predicted probability gives huge loss.

Convention 2: $y_i \in \{-1, +1\}$ (Margin-Based)

$$\ell_i(\theta) = \log(1 + e^{-y_i \theta^T x_i})$$
$y_i \in \{-1,+1\}$
The label is $-1$ ("negative") or $+1$ ("positive").
$y_i \theta^T x_i$
The margin. When the model agrees with the label (both positive or both negative), the margin is positive and the loss is small. When the model disagrees, the margin is negative and the loss is large.
$\log(1 + e^{-\text{margin}})$
A single, clean formula. No if/else, no separate cases. The margin does all the work.

Why the margin-based form works

The product $y_i \theta^T x_i$ is the key insight. Think of it this way:

  • When $y_i = +1$ and $\theta^T x_i$ is large and positive (correct, confident prediction): the margin $y_i \theta^T x_i$ is large and positive, so $e^{-\text{large positive}} \approx 0$, and $\ell_i \approx \log(1 + 0) = 0$. Almost no loss.
  • When $y_i = +1$ but $\theta^T x_i$ is negative (wrong prediction): the margin is negative, so $e^{-\text{negative}} = e^{\text{positive}}$ is large, and $\ell_i$ is large. Big penalty.
  • The same logic works symmetrically when $y_i = -1$: the model should produce a negative $\theta^T x_i$, which makes $y_i \theta^T x_i = (-1)(\text{negative}) = \text{positive}$, giving low loss.

Course convention: This course uses Convention 2 (margin-based, $y \in \{-1,+1\}$) throughout. All formulas below follow this convention. The two conventions produce mathematically equivalent optimization problems; the margin-based form is simply more compact.

PART D

The Objective Function

Analogy: the average regret. Each data point contributes a "regret" score based on how badly the model predicted its label. A correctly classified point with high confidence contributes almost zero regret. A misclassified point contributes a lot. The objective function is just the average regret across all data points; we want to find the parameters $\theta$ that minimize the collective regret.

Logistic Regression Objective (Margin-Based)

$$f(\theta) = \frac{1}{n}\sum_{i=1}^{n} \log(1 + e^{-y_i \theta^T x_i})$$
$n$
Number of data points.
$y_i \in \{-1, +1\}$
The label for the $i$-th data point.
$x_i \in \mathbb{R}^d$
The feature vector for the $i$-th data point (a column vector with $d$ entries).
$\theta \in \mathbb{R}^d$
The parameter vector we are trying to find. These $d$ numbers define the decision boundary.
$\theta^T x_i$
The raw score for data point $i$. This is the same linear combination as in linear regression: $\theta_1 x_{i1} + \theta_2 x_{i2} + \cdots + \theta_d x_{id}$.
$\frac{1}{n}\sum$
The average over all data points. This scaling does not affect where the minimum is, but it makes the gradient independent of $n$.

Three key properties from the optimization perspective

  1. Convex. The function $\log(1 + e^{-z})$ is convex in $z$, and each $z_i = y_i\theta^Tx_i$ is a linear (hence convex) function of $\theta$. The composition of a convex function with a linear function is convex. A sum of convex functions is convex. So $f$ is convex. Every local minimum is a global minimum (recall from Page 05).
  2. Smooth ($C^\infty$). The sigmoid and the exponential are infinitely differentiable. So the gradient, Hessian, and all higher derivatives exist everywhere. This means Newton's method (Page 07), BFGS (Page 12), and nonlinear CG (Page 11) all apply.
  3. Not strongly convex (without regularization). The Hessian can have zero eigenvalues if some data directions are uninformative. This means the convergence rate depends on the data, and gradient descent may be slow. Adding an $\ell_2$ penalty $\frac{\lambda}{2}\|\theta\|^2$ to the objective makes it strongly convex, guaranteeing a unique minimizer and faster convergence.

Comparison with linear regression (Page 23): The least-squares objective $\|A\theta - b\|^2$ is also convex and smooth, but it is a quadratic, so Newton converges in one step and the Hessian is constant. The logistic objective is not quadratic; the Hessian changes at every point. So Newton requires multiple iterations, and BFGS approximates the changing Hessian instead of computing it exactly.

PART E

The Gradient

Analogy: a classroom attention filter. Imagine a teacher grading a stack of exams. For each student, the gradient asks: "How much would adjusting the answer key change this student's score?" Students who clearly passed (high confidence, correct) barely register; their contribution to the gradient is almost zero. Students near the pass/fail boundary, or students who were misclassified, contribute the most. The gradient acts like an attention filter that focuses the optimization on the hard cases and ignores the easy ones.

Gradient of the Logistic Objective

$$\nabla f(\theta) = -\frac{1}{n}\sum_{i=1}^{n} \frac{y_i x_i}{1 + e^{y_i \theta^T x_i}}$$
$y_i x_i$
The "signed data point." When $y_i = +1$, this is just $x_i$. When $y_i = -1$, this is $-x_i$. The label flips the direction of the contribution.
$\frac{1}{1 + e^{y_i \theta^T x_i}}$
The weight for data point $i$. This is $\sigma(-y_i\theta^Tx_i)$: the sigmoid of the negative margin. When the margin is large and positive (correct, confident), this weight is close to 0 (the point is "handled"). When the margin is small or negative (uncertain or wrong), this weight is close to 1 (the point demands attention).
$-\frac{1}{n}\sum$
Negative sign because we are minimizing. The average makes the gradient scale-independent of $n$.

There is also a compact matrix form. Stack all data into a matrix $A \in \mathbb{R}^{n \times d}$ (rows are data points) and labels into $y \in \mathbb{R}^n$:

Matrix Form of the Gradient

$$\nabla f(\theta) = -\frac{1}{n}A^T\bigl(y \odot \sigma(-y \odot (A\theta))\bigr)$$
$A\theta \in \mathbb{R}^n$
The vector of raw scores for all $n$ data points.
$y \odot (A\theta)$
Elementwise product: the vector of margins $[y_1\theta^Tx_1, \ldots, y_n\theta^Tx_n]^T$.
$\sigma(-y \odot (A\theta))$
Apply the sigmoid to the negative margins. This gives the weight vector.
$y \odot \sigma(\cdot)$
Elementwise product with labels again, producing the signed weights.
$A^T(\cdot)$
Projects the $n$-dimensional weight vector back into the $d$-dimensional parameter space (same role as $A^T$ in the linear regression gradient).
$\odot$
Elementwise (Hadamard) product: $(u \odot v)_i = u_i v_i$.

Exam Simplification: Gradient at $\theta = 0$

At $\theta = 0$, every margin $y_i\theta^Tx_i = 0$, so $e^{y_i\theta^Tx_i} = e^0 = 1$ for all $i$. The gradient simplifies to:

$$\nabla f(0) = -\frac{1}{n}\sum_{i=1}^{n} \frac{y_i x_i}{1 + 1} = -\frac{1}{2n}\sum_{i=1}^{n} y_i x_i$$

This is a common exam starting point: "compute the gradient at the origin." Every data point gets equal weight ($\frac{1}{2}$), and the gradient is just the (scaled, negated) sum of the signed data vectors.

PART F

Solving with Optimization Methods

Analogy: choosing your hiking gear. Gradient descent (Page 06) is like hiking downhill with just your feet; slow but reliable, and you only need to feel the slope under your boots. BFGS (Page 12) is like having a terrain map that updates with every step; much faster because it learns the curvature of the landscape as it goes. Nonlinear CG (Page 11) is somewhere in between; it remembers which direction it came from and avoids zigzagging back and forth across a valley. All three methods work here because the logistic objective is smooth and convex.

Three Methods for Logistic Regression

Gradient Descent BFGS Nonlinear CG
Update Rule $\theta_{k+1} = \theta_k - \alpha_k \nabla f$ $\theta_{k+1} = \theta_k - \alpha_k H_k \nabla f$ $\theta_{k+1} = \theta_k + \alpha_k d_k$
Convergence Linear Superlinear Superlinear
Storage $O(d)$ $O(d^2)$ $O(d)$
Best When Large $n$ (use SGD variant) Moderate $d$ Large $d$, memory limited

Line search requirements: BFGS needs a Wolfe line search to maintain the condition $s^T y \gt 0$ that keeps the approximate Hessian positive definite (recall from Page 12). Nonlinear CG needs the Strong Wolfe conditions to ensure the conjugate direction does not degrade (recall from Page 11). Gradient descent works with any line search or even a well-chosen constant step size.

Why not Newton's method?

Newton's method (Page 07) would also work here, since the logistic objective is smooth and convex. But unlike linear regression where the Hessian is constant, the logistic Hessian changes at every iteration and costs $O(nd^2)$ to compute. BFGS approximates the Hessian from gradient information alone, achieving near-Newton convergence speed without ever forming the true Hessian. For moderate $d$, BFGS is typically the best practical choice.

The decision boundary is the set of points where the model is perfectly undecided:

Decision Boundary

$$\theta^T x = 0$$

This is a hyperplane in $\mathbb{R}^d$. Points with $\theta^T x \gt 0$ are classified as $+1$; points with $\theta^T x \lt 0$ are classified as $-1$; points exactly on the boundary have $\sigma(\theta^Tx) = 0.5$. In 2D, the decision boundary is a straight line. In 3D, it is a plane.

PART G

Worked Hand Computation

Analogy: calibrating the thermometer by hand. We are going to walk through one gradient descent step on a tiny dataset, computing every number explicitly. This is the kind of computation that appears on exams: small enough to do by hand, but it exercises every piece of the logistic regression formula.

One Gradient Descent Step for Logistic Regression

Given $n = 3$ data points with $d = 2$ features:

$x_1 = \begin{bmatrix}1\\0\end{bmatrix},\; y_1 = +1$;   $x_2 = \begin{bmatrix}0\\1\end{bmatrix},\; y_2 = -1$;   $x_3 = \begin{bmatrix}1\\1\end{bmatrix},\; y_3 = +1$

Starting at $\theta_0 = \begin{bmatrix}0\\0\end{bmatrix}$, learning rate $\eta = 1$. Compute one gradient descent step.

Step 1: Compute the margins $z_i = y_i \theta_0^T x_i$

Since $\theta_0 = \begin{bmatrix}0\\0\end{bmatrix}$, every dot product $\theta_0^T x_i = 0$.

$$z_1 = (+1)(0) = 0, \quad z_2 = (-1)(0) = 0, \quad z_3 = (+1)(0) = 0$$
All margins are zero. The model is completely undecided about every point.
Step 2: Compute the current loss $f(\theta_0)$
$$f(\theta_0) = \frac{1}{3}\sum_{i=1}^{3} \log(1 + e^{-z_i}) = \frac{1}{3} \cdot 3 \cdot \log(1 + e^0) = \frac{1}{3} \cdot 3 \cdot \log(2) = \log 2 \approx 0.693$$
$f(\theta_0) = \log 2 \approx 0.693$. This is the maximum possible average loss per point (since $\sigma(0) = 0.5$ for every point).
Step 3: Compute the gradient $\nabla f(\theta_0)$

Using the exam simplification from Part E (all margins are zero, so each weight is $\frac{1}{2}$):

$$\nabla f(\theta_0) = -\frac{1}{2n}\sum_{i=1}^{3} y_i x_i = -\frac{1}{6}\left[y_1 x_1 + y_2 x_2 + y_3 x_3\right]$$

Substituting each term:

$$= -\frac{1}{6}\left[(+1)\begin{bmatrix}1\\0\end{bmatrix} + (-1)\begin{bmatrix}0\\1\end{bmatrix} + (+1)\begin{bmatrix}1\\1\end{bmatrix}\right]$$ $$= -\frac{1}{6}\begin{bmatrix}1 + 0 + 1 \\ 0 - 1 + 1\end{bmatrix} = -\frac{1}{6}\begin{bmatrix}2\\0\end{bmatrix} = \begin{bmatrix}-1/3\\0\end{bmatrix}$$
$\nabla f(\theta_0) = \begin{bmatrix}-1/3\\0\end{bmatrix}$. The gradient points in the negative $\theta_1$ direction; the optimization wants to increase $\theta_1$.
Step 4: Update $\theta$
$$\theta_1 = \theta_0 - \eta \cdot \nabla f(\theta_0) = \begin{bmatrix}0\\0\end{bmatrix} - 1 \cdot \begin{bmatrix}-1/3\\0\end{bmatrix} = \begin{bmatrix}1/3\\0\end{bmatrix}$$
$\theta_1 = \begin{bmatrix}1/3\\0\end{bmatrix}$
Step 5: Interpret the new decision boundary

The decision boundary is $\theta^T x = 0$, which gives $\frac{1}{3}x_1 + 0 \cdot x_2 = 0$, i.e., $x_1 = 0$.

  • Points with $x_1 \gt 0$ are classified as $+1$: this correctly classifies $x_1 = (1,0)$ and $x_3 = (1,1)$.
  • Points with $x_1 \lt 0$ are classified as $-1$: $x_2 = (0,1)$ has $x_1 = 0$, so it lands exactly on the boundary (uncertain).
After just one step, the model has learned that the first feature matters for classification, but it has not yet figured out how to handle the second feature.

Sanity check: the gradient had a nonzero first component ($-1/3$) and zero second component. This makes sense; looking at the labels, positive points ($x_1$ and $x_3$) both have $x_1 = 1$, while the negative point ($x_2$) has $x_1 = 0$. Feature 1 carries class-separating information. Feature 2 is present in both a positive ($x_3$) and a negative ($x_2$) point, so its contribution cancels in the gradient.

GLOSSARY

Key Terms

Sigmoid ($\sigma$)
The function $\sigma(z) = \frac{1}{1+e^{-z}}$ that maps any real number to the interval $(0, 1)$. Used to convert a raw score into a probability. Also called the logistic function or the squashing function.
Logit
The raw score $z = \theta^Tx$ before the sigmoid is applied. The logit can be any real number. It is the "inverse" of the sigmoid: $z = \log\frac{p}{1-p}$ where $p = \sigma(z)$. The name "logistic regression" comes from "log-odds" → "logit."
Logistic Loss
The per-sample loss $\ell_i = \log(1 + e^{-y_i\theta^Tx_i})$ (margin form) or $\ell_i = -[y_i\log p_i + (1-y_i)\log(1-p_i)]$ (probabilistic form). Both are convex in $\theta$. Also called log-loss.
Cross-Entropy
Another name for the probabilistic form of the logistic loss (Convention 1). The name comes from information theory: it measures the "distance" between the true label distribution and the predicted probability distribution. Minimizing cross-entropy is equivalent to maximum likelihood estimation.
Margin
The product $y_i \theta^T x_i$. Positive margin means the model agrees with the true label; negative margin means it disagrees. Larger positive margin means more confident correct prediction. The margin-based loss $\log(1+e^{-\text{margin}})$ penalizes negative margins heavily and rewards large positive margins with near-zero loss.
Decision Boundary
The set $\{\,x : \theta^T x = 0\,\}$, a hyperplane in $\mathbb{R}^d$. Points on one side are classified as $+1$, points on the other as $-1$. In 2D it is a line; in 3D it is a plane. Logistic regression always produces a linear decision boundary (unlike SVMs with kernels).
Convex (Not Strongly Convex)
The logistic objective is convex (every local min is global; recall Page 05), but without regularization it is not strongly convex: the Hessian can have zero eigenvalues. Strong convexity can be enforced by adding an $\ell_2$ penalty $\frac{\lambda}{2}\|\theta\|^2$, which guarantees a unique minimizer and improves convergence rates.
Support (Connection to SVM)
In logistic regression, all data points contribute to the gradient (some with tiny weight). In a support vector machine (Page 25), only the points near the decision boundary (the "support vectors") matter at all. SVM uses a hinge loss instead of the smooth logistic loss. The two approaches produce similar decision boundaries but have different mathematical properties.
