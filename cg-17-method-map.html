Part A: The Decision Flowchart; Which Method Do I Use?

Physical analogy; A mechanic's diagnostic tree.

When a car won't start, a good mechanic doesn't randomly try parts. They follow a decision tree: is the engine cranking? If yes, check fuel. If no, check battery. Each branch narrows the problem until you land on the right fix. This flowchart works the same way: start with "what kind of problem do I have?" and follow the branches until you land on the right method.

Below is the master decision flowchart for the entire course. Every method we studied fits somewhere in this tree. Follow the arrows from the top. Each colored box links to the page where that method is fully explained.

How to use this chart on the exam: When you see a problem, first classify it; unconstrained? constrained? system of equations? linear system? Then follow the appropriate branch. The information you are given (gradient, Hessian, etc.) determines which method is appropriate.
Classification / ML methods (Pages 24, 25, 26): Three application areas feed directly into this flowchart. Logistic Regression (Page 24) is unconstrained optimization of convex logistic loss, solved with GD, BFGS, or Nonlinear CG. SVM (Page 25) is a constrained convex QP, solved via the dual problem and KKT conditions. MLP / Neural Networks (Page 26) are non-convex unconstrained optimization, solved with gradient descent using backpropagation for gradients. All three reuse the core methods shown above.

Part C: The Three Big Design Choices

Every optimization algorithm we studied makes three choices. Understanding these choices is the thread that ties everything together.

Choice 1: Search Direction; "Which way do I walk?"

At each iteration, you pick a direction $d_k$ to move in. Every method differs in how it picks this direction:

  • Steepest descent ($d_k = -\nabla f_k$): Simple, cheap, but zigzags on elongated valleys. Page 06
  • Newton direction ($d_k = -[\nabla^2 f_k]^{-1}\nabla f_k$): Uses curvature info for a much better direction. Expensive but fast convergence. Page 07
  • Conjugate gradient direction: Like GD but remembers past directions, avoids revisiting. Cheap memory, good convergence. Page 10, Page 11
  • Quasi-Newton direction ($d_k = -H_k \nabla f_k$): Approximates Newton without computing Hessians. Best of both worlds for medium-scale. Page 12, Page 13

Choice 2: Globalization; "How do I make sure I actually converge?"

Newton and quasi-Newton methods only guarantee convergence locally (near $x^*$). To converge from any starting point, we need a globalization strategy:

  • Line search: Pick direction first, then find how far to go along that direction. Test conditions (Armijo, Wolfe) to ensure sufficient progress. Page 08
  • Trust region: Pick a radius first (how far you trust your model), then find the best direction within that radius. Adjust radius based on how well the model predicted reality. Page 09

Key tradeoff: Line search is simpler to implement and pairs naturally with BFGS (since Wolfe conditions give you the curvature condition for free). Trust region is more robust (handles indefinite Hessians, never needs the direction to be a descent direction) and pairs naturally with SR1.

Choice 3: Step Size; "How far do I walk?"

For line search methods, the step size $\alpha_k$ matters enormously:

  • Exact line search: Minimize $\phi(\alpha) = f(x_k + \alpha d_k)$ exactly. Only practical for quadratics (closed-form). Page 06
  • Backtracking (Armijo only): Start with $\alpha = 1$, shrink by factor $\rho$ until Armijo holds. Simple, cheap, good enough for GD. Page 08
  • Wolfe conditions: Armijo + curvature condition. More expensive search, but essential for BFGS (guarantees $s^T y \gt 0$). Page 08
  • Strong Wolfe: Tighter curvature condition. Needed for nonlinear CG. Page 08
Putting it all together; the hiking analogy.

Think of optimization as hiking down a mountain to the valley floor:

Direction: GD says "go straight downhill." Newton says "I calculated the shape of the valley floor; head toward it." CG says "don't retrace steps; each new direction finishes what previous ones started." BFGS says "I've been tracking the curvature as I walk; I have a pretty good mental model of the terrain."

Globalization: Line search says "I'll pick a direction, then carefully decide how far to walk before stopping." Trust region says "I only trust my map within a certain radius; find the best step inside that circle."

Step size: Exact line search says "walk to the absolute bottom of the slope in this direction." Armijo says "walk until I've made enough progress." Wolfe says "walk until I've made enough progress AND the slope has flattened enough."

Part E: Key Connections Between Methods

These connections are the conceptual glue of the entire course. If you understand these four relationships, you understand how optimization methods form a single coherent story rather than a bag of disconnected algorithms.

Connection 1: Newton for Optimization IS Newton for Equations applied to the gradient

Newton's method for optimization (Page 07) solves $\nabla f(x) = 0$. But $\nabla f(x) = 0$ is just a system of nonlinear equations! So Newton for optimization is literally Newton for nonlinear systems (Page 14) applied to $r(x) = \nabla f(x)$.

The Jacobian of $r(x) = \nabla f(x)$ is $J(x) = \nabla^2 f(x)$. So the Newton step $J(x_k) p_k = -r(x_k)$ becomes $\nabla^2 f(x_k) p_k = -\nabla f(x_k)$; exactly the Newton step for optimization.

Connection 2: BFGS approximates Newton without computing Hessians

Newton needs $\nabla^2 f$ at every step; expensive or impossible for large problems. BFGS (Page 12) builds an approximation to the inverse Hessian by watching how the gradient changes from step to step. It uses the secant condition: $B_{k+1}s_k = y_k$, where $s_k$ is the step and $y_k$ is the gradient change.

The beautiful result: BFGS achieves superlinear convergence (almost as fast as Newton) with only $O(n^2)$ cost per step instead of $O(n^3)$. The trade: quadratic convergence drops to superlinear, but the per-step cost drops enormously.

Connection 3: CG is like GD but with memory

Gradient descent (Page 06) picks the steepest direction at each step, completely ignoring what it did before. This causes zigzagging. Conjugate gradient (Page 10) fixes this by remembering the previous direction and making sure the new direction is "conjugate" to all previous ones.

"Conjugate" means the new direction finishes what the old direction started, without undoing any progress. On an $n$-dimensional quadratic, this guarantees convergence in exactly $n$ steps. GD might take thousands of zigzag steps on the same problem.

Connection 4: Trust region and line search are two globalization philosophies

Both serve the same purpose: make Newton (or any method) converge from far away, not just near the solution.

Line search (Page 08): "I've chosen a direction. Now let me figure out how far to go." Direction first, then step size.

Trust region (Page 09): "I only trust my model within radius $\Delta$. Let me find the best step inside that circle." Step size first (the radius), then direction.

Trust region is more robust because it can handle indefinite Hessians (the Cauchy point always exists). Line search requires a descent direction, which needs a positive definite Hessian or a modification.

Connection 5: Broyden is to nonlinear systems what BFGS is to optimization

Broyden's method (Page 14) approximates the Jacobian for systems $r(x) = 0$, just like BFGS (Page 12) approximates the Hessian for $\min f(x)$. Both use secant-type conditions and rank-one or rank-two updates. The intellectual DNA is the same.

Connection 6: KKT conditions unify unconstrained and constrained optimization

For unconstrained problems, the optimality condition is $\nabla f(x^*) = 0$ (Page 04). For constrained problems, the KKT conditions (Page 15) generalize this: $\nabla f(x^*) + \sum \lambda_i \nabla c_i(x^*) = 0$. When there are no constraints, the sum disappears and you recover $\nabla f = 0$. KKT is the "master optimality condition."

Exam Tip: Connection questions

If the exam asks "explain how method X relates to method Y," the answer is usually one of the six connections above. The most common: "How does BFGS relate to Newton?" (Connection 2) and "How does Newton for optimization relate to Newton for equations?" (Connection 1).

Part F: Classification and ML Methods

These three methods show how the optimization toolkit from the entire course powers real machine learning applications. Each one maps directly back to a problem type and solver we already studied.

Logistic Regression (Page 24): Unconstrained Optimization for Binary Classification

Objective: Minimize the logistic loss function, which is convex and smooth. Given data points $(x_i, y_i)$ with $y_i \in \{0, 1\}$, find weights $w$ that minimize:

$$\min_w \; -\sum_{i=1}^{m} \bigl[ y_i \ln(\sigma(w^T x_i)) + (1 - y_i) \ln(1 - \sigma(w^T x_i)) \bigr]$$

Solved with: Gradient Descent (Page 06), BFGS (Page 12), or Nonlinear CG (Page 11). Because the loss is convex and smooth, all our unconstrained methods apply directly.

Course connection: This is a straight application of the unconstrained branch of the decision flowchart above. The only new ingredient is the logistic loss function; the solvers are ones we already know.

SVM (Page 25): Convex QP with Linear Inequality Constraints

Objective: Maximize the margin between two classes by minimizing $\frac{1}{2}\|w\|^2$ subject to $y_i(w^T x_i + b) \geq 1$ for all training points. This is a convex quadratic program (QP).

Solved with: The dual problem and KKT conditions. The KKT multipliers $\alpha_i$ identify the support vectors (the data points that sit exactly on the margin boundary).

Course connection: This uses constrained optimization (Page 15) and duality (Page 16). The KKT conditions from Connection 6 above are exactly what determine the optimal solution.

MLP / Neural Network (Page 26): Non-Convex Unconstrained Optimization

Objective: Minimize the MSE loss (or cross-entropy loss) over all network weights. For a network with layers $W_1, W_2, \ldots$, the loss is a highly non-convex function of all these weight matrices.

Solved with: Gradient descent, where the gradients are computed via backpropagation (the chain rule applied layer by layer). The non-convexity means we can get stuck in local minima or saddle points; there is no guarantee of finding the global optimum.

Course connection: This uses gradient descent (Page 06) and the chain rule for computing derivatives (Page 02). The key difference from everything else in the course: the loss landscape is non-convex, so convergence theory is weaker and practical tricks (learning rate schedules, momentum) become essential.

Physical analogy; three different uses of the same toolbox.

Think of the optimization methods from this course as a set of power tools. Logistic regression is like using those tools to build a straight fence (convex, well-behaved; the tools work perfectly). SVM is like building a fence with property-line constraints (you need to respect boundaries; KKT conditions enforce them). Neural networks are like sculpting a complex shape (the surface has bumps and valleys everywhere; you do your best with gradient descent, but there is no guarantee you found the absolute best shape).
Practical consideration: Stopping criteria for all iterative methods

Every iterative method in the table above needs a rule for when to stop. In practice, you check three conditions and stop when any one is satisfied:
  1. Gradient norm: $\|\nabla f(x_k)\| \lt \epsilon_g$. The first-order optimality condition says $\nabla f(x^*) = 0$, so a tiny gradient means you are near a stationary point.
  2. Step size: $\|x_{k+1} - x_k\| \lt \epsilon_x$. If the iterates are barely moving, further steps are unlikely to make meaningful progress.
  3. Function value change: $|f(x_{k+1}) - f(x_k)| \lt \epsilon_f$. If the objective is no longer decreasing appreciably, you have effectively converged.

A robust implementation checks all three. Relying on only one can be misleading; for example, the gradient can be small on a flat plateau far from the solution, and the function value can stall temporarily near a saddle point.
