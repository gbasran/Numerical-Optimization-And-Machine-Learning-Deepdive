Newton's Method for Nonlinear Equations

From finding square roots to solving systems of equations; one tangent line (or tangent plane) at a time

Part A: Newton's Method for Scalar Equations

The Tangent-Line Root-Finding Analogy

Imagine you have a curve drawn on a piece of paper and you need to find where it crosses the horizontal axis (the root). You cant solve the equation algebraically; the curve is too complicated. So heres the trick:

1. Pick a starting guess $x_0$ on the horizontal axis and look up to where the curve is at that point: the height is $f(x_0)$.
2. At that point on the curve, draw a tangent line; a straight line that just touches the curve and has the same slope as the curve there. The slope of that tangent line is $f'(x_0)$.
3. Find where your tangent line crosses zero. Thats your next guess, $x_1$.
4. Repeat: draw a new tangent line at $x_1$, find where it crosses zero to get $x_2$, and so on.

Why does this work? Because a tangent line is the best local straight-line approximation to the curve. Near the root, the tangent line and the curve are almost identical, so the tangent lines zero-crossing is very close to the curves zero-crossing. Each iteration the approximation gets better, and we converge quadratically; the number of correct digits roughly doubles every step.

We want to solve $f(x) = 0$ for some nonlinear function $f: \mathbb{R} \to \mathbb{R}$. The tangent line at $x_k$ is:

$$f(x_k) + f'(x_k)(x - x_k) = 0$$

Solving for $x$ gives us the next iterate:

Scalar Newton's Method $$x_{k+1} = x_k - \frac{f(x_k)}{f'(x_k)}$$
$x_{k+1}$
the next (improved) guess for the root
$x_k$
the current guess
$f(x_k)$
the value of the function at the current guess (how far from zero we are)
$f'(x_k)$
the derivative (slope) of the function at the current guess
$\frac{f(x_k)}{f'(x_k)}$
the correction; how far along the tangent line we need to go to hit zero
The formula says: "Take your current guess, then subtract the ratio of the function value to the derivative." If $f(x_k) > 0$ and the slope is positive, we move left. If $f(x_k) > 0$ and the slope is negative, we move right. The formula automatically figures out the correct direction.

Hand Computation: Finding $\sqrt{2}$ by Newton's Method

We want to find $\sqrt{2}$, which is the positive root of $f(x) = x^2 - 2 = 0$.

The derivative is $f'(x) = 2x$.

So the Newton iteration becomes:

$$x_{k+1} = x_k - \frac{x_k^2 - 2}{2x_k}$$
Iteration 0 → 1: starting at $x_0 = 1.5$
$f(x_0) = (1.5)^2 - 2 = 2.25 - 2 = 0.25$
$f'(x_0) = 2(1.5) = 3.0$
$x_1 = 1.5 - \frac{0.25}{3.0} = 1.5 - 0.08333... = 1.41\overline{6}$
Iteration 1 → 2: at $x_1 = 1.41\overline{6}$
$f(x_1) = (1.41\overline{6})^2 - 2 = 2.006944... - 2 = 0.006944...$
$f'(x_1) = 2(1.41\overline{6}) = 2.8\overline{3}$
$x_2 = 1.41\overline{6} - \frac{0.006944...}{2.8\overline{3}} = 1.41\overline{6} - 0.002450... = 1.414216...$
Iteration 2 → 3: at $x_2 = 1.414216...$
$f(x_2) = (1.414216...)^2 - 2 = 2.000007... - 2 = 0.000007...$
$f'(x_2) = 2(1.414216...) = 2.828432...$
$x_3 = 1.414216... - \frac{0.000007...}{2.828432...} = 1.414213562...$

After just 3 iterations: $x_3 = 1.41421356...$

True value: $\sqrt{2} = 1.41421356237...$. We have 9 correct digits after only 3 steps! Thats quadratic convergence in action; the number of correct digits roughly doubled each iteration (1 → 3 → 6 → 9+).

Part B: Newton's Method for Vector Systems

The Multi-Equation Intersection Analogy

In the scalar case, we had one equation $f(x) = 0$ and one unknown $x$. Picture a single curve, and you wanted to find where it crosses the $x$-axis. Now imagine you have two equations and two unknowns; like a circle and a line. Each equation defines a curve in the plane, and you need to find the intersection point where both curves cross simultaneously.

You cant just draw one tangent line anymore. Instead, at your current guess $(x_1, x_2)$, you approximate EACH curve by its tangent line (or tangent plane in higher dimensions). Where those two tangent lines intersect gives you the next guess. This is exactly what the vector version of Newton does.

For $n$ equations in $n$ unknowns, you have $n$ "surfaces" that all need to pass through zero simultaneously. The Jacobian matrix replaces the scalar derivative; it contains the slopes of ALL equations in ALL directions, so it can tell you where all the tangent planes meet.

We want to solve the system $\mathbf{r}(x) = \mathbf{0}$ where $\mathbf{r}: \mathbb{R}^n \to \mathbb{R}^n$ is a vector-valued function. That means:

$$\mathbf{r}(x) = \begin{bmatrix} r_1(x_1, x_2, \ldots, x_n) \\ r_2(x_1, x_2, \ldots, x_n) \\ \vdots \\ r_n(x_1, x_2, \ldots, x_n) \end{bmatrix} = \begin{bmatrix} 0 \\ 0 \\ \vdots \\ 0 \end{bmatrix}$$

The Jacobian matrix $J(x)$ plays the role that $f'(x)$ played for scalar equations. Recall from Page 02 (Multivariable Bridge) that the Jacobian is the matrix of all first partial derivatives:

The Jacobian Matrix $$J(x) = \begin{bmatrix} \frac{\partial r_1}{\partial x_1} & \frac{\partial r_1}{\partial x_2} & \cdots & \frac{\partial r_1}{\partial x_n} \\[6pt] \frac{\partial r_2}{\partial x_1} & \frac{\partial r_2}{\partial x_2} & \cdots & \frac{\partial r_2}{\partial x_n} \\[6pt] \vdots & \vdots & \ddots & \vdots \\[6pt] \frac{\partial r_n}{\partial x_1} & \frac{\partial r_n}{\partial x_2} & \cdots & \frac{\partial r_n}{\partial x_n} \end{bmatrix}$$

Row $i$ contains the partial derivatives of the $i$-th equation $r_i$ with respect to each variable. Column $j$ contains the partial derivatives of all equations with respect to $x_j$.

Why Does the Jacobian Replace $f'$?

For a single equation $f(x) = 0$, the derivative $f'(x_k)$ tells you the slope of the function at $x_k$. It is a single number. Newton uses it to build a linear approximation: $f(x) \approx f(x_k) + f'(x_k)(x - x_k)$.

For a system of $n$ equations, the Jacobian $J(x_k)$ is an $n \times n$ matrix that tells you the slope of EVERY equation in EVERY direction. It builds a linear approximation of the entire system: $\mathbf{r}(x) \approx \mathbf{r}(x_k) + J(x_k)(x - x_k)$.

Setting this linear approximation to zero and solving for the step $p_k = x - x_k$ gives Newton's method for systems.

Newton's Method for Nonlinear Systems
Given: starting point $x^{(0)}$, tolerance $\epsilon$
For $k = 0, 1, 2, \ldots$
1. Evaluate $\mathbf{r}(x^{(k)})$ ← how far are we from solving the system?
2. If $\|\mathbf{r}(x^{(k)})\| < \epsilon$, STOP. Solution found. ← close enough to zero
3. Evaluate $J(x^{(k)})$ ← build the Jacobian at current point
4. Solve $J(x^{(k)}) \, p^{(k)} = -\mathbf{r}(x^{(k)})$ for $p^{(k)}$ ← find the Newton step
5. Update: $x^{(k+1)} = x^{(k)} + p^{(k)}$ ← take the step
Key point: Step 4 says "solve" the linear system, NOT "compute the inverse." We never compute $J^{-1}$ explicitly; we solve the linear system $Jp = -\mathbf{r}$ using Gaussian elimination (or LU factorization). This is cheaper and more numerically stable.

Part C: Full Hand Computation; Mirrors Quiz 7

EXAM ALERT: Quiz 7 Problem Type

This is the exact type of computation you saw on Quiz 7. You are given a nonlinear system, a starting point, and asked to perform one or two Newton iterations. You must: (1) write the residual vector, (2) compute the Jacobian, (3) evaluate both at the current point, (4) solve the linear system for the step, (5) update. Show every intermediate calculation.

The Problem: Find the intersection of the unit circle and the line $y = x$. That is, solve:

$$r_1(x_1, x_2) = x_1^2 + x_2^2 - 1 = 0 \quad \text{(unit circle)}$$ $$r_2(x_1, x_2) = x_1 - x_2 = 0 \quad \text{(diagonal line } y = x \text{)}$$

starting from $x^{(0)} = \begin{bmatrix} 0.8 \\ 0.5 \end{bmatrix}$.

We know the true solution (in the first quadrant) is $\left(\frac{1}{\sqrt{2}}, \frac{1}{\sqrt{2}}\right) \approx (0.70711, 0.70711)$.

Newton Iteration 1: from $x^{(0)} = [0.8,\; 0.5]^T$

Step 1: Write the residual vector
$$\mathbf{r}(x) = \begin{bmatrix} x_1^2 + x_2^2 - 1 \\ x_1 - x_2 \end{bmatrix}$$
Step 2: Compute the Jacobian (partial derivatives)
$$J(x) = \begin{bmatrix} \frac{\partial r_1}{\partial x_1} & \frac{\partial r_1}{\partial x_2} \\[6pt] \frac{\partial r_2}{\partial x_1} & \frac{\partial r_2}{\partial x_2} \end{bmatrix} = \begin{bmatrix} 2x_1 & 2x_2 \\ 1 & -1 \end{bmatrix}$$

Lets verify each entry:

  • $\frac{\partial}{\partial x_1}(x_1^2 + x_2^2 - 1) = 2x_1$; only the $x_1^2$ term survives
  • $\frac{\partial}{\partial x_2}(x_1^2 + x_2^2 - 1) = 2x_2$; only the $x_2^2$ term survives
  • $\frac{\partial}{\partial x_1}(x_1 - x_2) = 1$
  • $\frac{\partial}{\partial x_2}(x_1 - x_2) = -1$
Step 3: Evaluate $\mathbf{r}$ and $J$ at $x^{(0)} = [0.8,\; 0.5]^T$
$$\mathbf{r}(x^{(0)}) = \begin{bmatrix} (0.8)^2 + (0.5)^2 - 1 \\ 0.8 - 0.5 \end{bmatrix} = \begin{bmatrix} 0.64 + 0.25 - 1 \\ 0.3 \end{bmatrix} = \begin{bmatrix} -0.11 \\ 0.3 \end{bmatrix}$$ $$J(x^{(0)}) = \begin{bmatrix} 2(0.8) & 2(0.5) \\ 1 & -1 \end{bmatrix} = \begin{bmatrix} 1.6 & 1.0 \\ 1 & -1 \end{bmatrix}$$
Step 4: Solve $J(x^{(0)})\,p^{(0)} = -\mathbf{r}(x^{(0)})$ for $p^{(0)}$

We need to solve:

$$\begin{bmatrix} 1.6 & 1.0 \\ 1 & -1 \end{bmatrix} \begin{bmatrix} p_1 \\ p_2 \end{bmatrix} = -\begin{bmatrix} -0.11 \\ 0.3 \end{bmatrix} = \begin{bmatrix} 0.11 \\ -0.3 \end{bmatrix}$$

Method: 2×2 inverse formula. For $\begin{bmatrix} a & b \\ c & d \end{bmatrix}^{-1} = \frac{1}{ad - bc}\begin{bmatrix} d & -b \\ -c & a \end{bmatrix}$:

Determinant: $\det(J) = (1.6)(-1) - (1.0)(1) = -1.6 - 1.0 = -2.6$

$$J^{-1} = \frac{1}{-2.6}\begin{bmatrix} -1 & -1.0 \\ -1 & 1.6 \end{bmatrix} = \begin{bmatrix} 0.38462 & 0.38462 \\ 0.38462 & -0.61538 \end{bmatrix}$$

Now compute $p^{(0)} = J^{-1} \cdot \begin{bmatrix} 0.11 \\ -0.3 \end{bmatrix}$:

$$p_1 = (0.38462)(0.11) + (0.38462)(-0.3) = 0.04231 - 0.11538 = -0.07308$$ $$p_2 = (0.38462)(0.11) + (-0.61538)(-0.3) = 0.04231 + 0.18462 = 0.22692$$ $$p^{(0)} = \begin{bmatrix} -0.07308 \\ 0.22692 \end{bmatrix}$$

Alternative: Gaussian elimination (verify).

From Row 2: $p_1 - p_2 = -0.3$, so $p_1 = p_2 - 0.3$.

Substitute into Row 1: $1.6(p_2 - 0.3) + 1.0 \cdot p_2 = 0.11$

$1.6 p_2 - 0.48 + p_2 = 0.11$

$2.6 p_2 = 0.59$

$p_2 = 0.59 / 2.6 = 0.22692$

$p_1 = 0.22692 - 0.3 = -0.07308$ ✓

Step 5: Update $x^{(1)} = x^{(0)} + p^{(0)}$
$$x^{(1)} = \begin{bmatrix} 0.8 \\ 0.5 \end{bmatrix} + \begin{bmatrix} -0.07308 \\ 0.22692 \end{bmatrix} = \begin{bmatrix} 0.72692 \\ 0.72692 \end{bmatrix}$$

After 1 iteration: $x^{(1)} = [0.72692,\; 0.72692]^T$

True solution: $[0.70711, 0.70711]^T$. We went from errors of $[0.093, -0.207]$ to errors of $[0.020, 0.020]$. Notice $x_1 = x_2$ already; Newton nailed the constraint $x_1 = x_2$ in one step because that equation is already linear!

Newton Iteration 2: from $x^{(1)} = [0.72692,\; 0.72692]^T$

Step 1: Evaluate $\mathbf{r}$ at $x^{(1)}$
$$\mathbf{r}(x^{(1)}) = \begin{bmatrix} (0.72692)^2 + (0.72692)^2 - 1 \\ 0.72692 - 0.72692 \end{bmatrix} = \begin{bmatrix} 0.52841 + 0.52841 - 1 \\ 0 \end{bmatrix} = \begin{bmatrix} 0.05683 \\ 0 \end{bmatrix}$$
Step 2: Evaluate $J$ at $x^{(1)}$
$$J(x^{(1)}) = \begin{bmatrix} 2(0.72692) & 2(0.72692) \\ 1 & -1 \end{bmatrix} = \begin{bmatrix} 1.45385 & 1.45385 \\ 1 & -1 \end{bmatrix}$$
Step 3: Solve $J(x^{(1)})\,p^{(1)} = -\mathbf{r}(x^{(1)})$
$$\begin{bmatrix} 1.45385 & 1.45385 \\ 1 & -1 \end{bmatrix} \begin{bmatrix} p_1 \\ p_2 \end{bmatrix} = \begin{bmatrix} -0.05683 \\ 0 \end{bmatrix}$$

From Row 2: $p_1 - p_2 = 0$, so $p_1 = p_2$.

Substitute into Row 1: $1.45385 \cdot p_1 + 1.45385 \cdot p_1 = -0.05683$

$2.90769 \cdot p_1 = -0.05683$

$p_1 = -0.01954$

$p_2 = -0.01954$

Step 4: Update
$$x^{(2)} = \begin{bmatrix} 0.72692 \\ 0.72692 \end{bmatrix} + \begin{bmatrix} -0.01954 \\ -0.01954 \end{bmatrix} = \begin{bmatrix} 0.70738 \\ 0.70738 \end{bmatrix}$$

After 2 iterations: $x^{(2)} = [0.70738,\; 0.70738]^T$

True solution: $[0.70711, 0.70711]^T$. Error dropped from $0.020$ to $0.00027$; thats quadratic convergence! The error squared from one step to the next.

Part D: Inexact Newton and Broyden's Method

Inexact Newton Methods

The "Good Enough Directions" Analogy

In exact Newton, we solve $J(x_k)p_k = -\mathbf{r}(x_k)$ precisely. For small systems (2×2, 3×3), thats cheap. But if you have 10,000 equations in 10,000 unknowns, solving a 10,000×10,000 linear system exactly is expensive.

Inexact Newton says: "Dont bother solving perfectly. Get an approximately correct step using an iterative solver like Conjugate Gradient (see Page 10)." As long as the approximate solution is good enough (controlled by a "forcing term" $\eta_k$), Newton still converges.
Inexact Newton Condition

Instead of solving $J_k p_k = -\mathbf{r}_k$ exactly, find $p_k$ such that:

$$\|J_k p_k + \mathbf{r}_k\| \leq \eta_k \|\mathbf{r}_k\|$$
$\eta_k$
the "forcing term"; controls how accurately we solve the linear system. $\eta_k \in (0, 1)$.
$\eta_k$ small
solve more accurately; closer to exact Newton, faster convergence but more work per step
$\eta_k$ large
solve less accurately; cheaper per step but slower convergence

Common strategy: start with large $\eta_k$ (cheap steps) and decrease it as you approach the solution (more accurate steps when they matter most).

Broyden's Method

The "Learning the Jacobian" Analogy

Computing the full Jacobian at every step is expensive; you need to evaluate all $n^2$ partial derivatives. Broyden says: "Start with the true Jacobian (or an approximation), then UPDATE it cheaply at each step based on what you learn from the function values."

This is the exact same philosophy as BFGS (Page 12): BFGS approximates the Hessian for optimization; Broyden approximates the Jacobian for nonlinear equations. Both learn from the history of steps and function/gradient changes.
Broyden's Update Formula $$J_{k+1} = J_k + \frac{(\mathbf{r}_{k+1} - \mathbf{r}_k - J_k s_k)\,s_k^T}{s_k^T s_k}$$
$J_k$
current approximate Jacobian (an $n \times n$ matrix)
$J_{k+1}$
updated approximate Jacobian
$s_k = x_{k+1} - x_k$
the step we just took
$\mathbf{r}_{k+1} - \mathbf{r}_k$
how the residual changed (the "observed change")
$J_k s_k$
how the current approximation PREDICTED the residual would change
$\mathbf{r}_{k+1} - \mathbf{r}_k - J_k s_k$
the "prediction error"; the difference between observed and predicted

The update adds a correction term that fixes the prediction error in the direction $s_k$. This is a rank-1 update (compare with SR1 on Page 13).

Broyden is to Nonlinear Equations What BFGS is to Optimization

Both methods share the same core idea: avoid computing expensive second-order information (the Hessian in optimization, the Jacobian in root-finding) by learning it incrementally from the history of steps and function evaluations.

Part E: Connection to Optimization

The "Nonlinear Equations IS Optimization" Connection

Here is one of the deepest connections in this entire course. When we do optimization, we find stationary points by solving $\nabla f(x) = 0$. But "$\nabla f(x) = 0$" is itself a system of nonlinear equations! The gradient $\nabla f$ plays the role of $\mathbf{r}$, and the Hessian $\nabla^2 f$ plays the role of $J$.
The Dictionary Between the Two Worlds
Nonlinear EquationsOptimization
Residual $\mathbf{r}(x)$Gradient $\nabla f(x)$
Jacobian $J(x)$Hessian $\nabla^2 f(x)$
Newton step: solve $Jp = -\mathbf{r}$Newton step: solve $\nabla^2 f \cdot p = -\nabla f$ (from Page 07)
Broyden approximates $J$BFGS approximates $\nabla^2 f$ (from Page 12)
Inexact Newton (CG on $Jp = -\mathbf{r}$)Newton-CG (CG on Hessian system, from Page 10)

Why This Matters

Newton's method for optimization (which you learned on Page 07) is literally Newton's method for nonlinear equations applied to the system $\nabla f(x) = 0$. The Hessian IS the Jacobian of the gradient. Every idea from this page; inexact solves, quasi-Newton updates; has a direct counterpart in optimization. Understanding one gives you the other for free.

On the exam: If asked "what is the connection between Newton for equations and Newton for optimization?", the answer is: solving $\nabla f(x) = 0$ is a nonlinear system where $\mathbf{r} = \nabla f$ and $J = \nabla^2 f$. The Newton step $\nabla^2 f(x_k)\,p_k = -\nabla f(x_k)$ for optimization is exactly $J(x_k)\,p_k = -\mathbf{r}(x_k)$ for equations.

Glossary

Nonlinear System
A set of $n$ equations in $n$ unknowns where at least one equation is nonlinear: $\mathbf{r}(x) = \mathbf{0}$ with $\mathbf{r}: \mathbb{R}^n \to \mathbb{R}^n$.
Root / Zero
A point $x^*$ where $\mathbf{r}(x^*) = \mathbf{0}$. For a scalar equation, its where the curve crosses the $x$-axis.
Residual $\mathbf{r}(x)$
The vector of function values at $x$. We want to drive all components to zero simultaneously.
Jacobian $J(x)$
The $n \times n$ matrix of all first partial derivatives of $\mathbf{r}$. Entry $(i,j)$ is $\frac{\partial r_i}{\partial x_j}$. Generalizes the scalar derivative $f'(x)$. See Page 02.
Newton Step $p_k$
The solution to $J(x_k)\,p_k = -\mathbf{r}(x_k)$. Tells you how to update: $x_{k+1} = x_k + p_k$.
Quadratic Convergence
Near a solution, the error at step $k+1$ is proportional to the error at step $k$ squared: $\|e_{k+1}\| \leq C\|e_k\|^2$. The number of correct digits roughly doubles each step.
Inexact Newton
Solving $J_k p_k = -\mathbf{r}_k$ approximately (e.g., using CG) instead of exactly. Controlled by the forcing term $\eta_k$.
Broyden's Method
A quasi-Newton method for nonlinear equations that approximates the Jacobian using rank-1 updates from step history, analogous to how BFGS approximates the Hessian.
Forcing Term $\eta_k$
Controls the accuracy of the inexact Newton solve: $\|J_k p_k + \mathbf{r}_k\| \leq \eta_k \|\mathbf{r}_k\|$. Smaller $\eta_k$ means more accurate (and more expensive) solves.
Superlinear Convergence
Faster than linear but not quite quadratic: the ratio $\|e_{k+1}\|/\|e_k\| \to 0$ as $k \to \infty$. Broyden achieves this.
